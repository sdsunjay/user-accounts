User Accounts Security Details

  • All information is transferred over HTTPS
    • Cryptographic protocols are TLSv1 TLSv1.1 TLSv1.2 (No version of SSL is allow due to this vulnerability)
    • Cryptographic ciphers suggested here are used
  • Client side encryption works as well (see it here, but it has yet to be integrated into production)
    • Passwords are stored in a MySQL Database
      • Passwords are encrypted using the CRYPT_BLOWFISH algorithm
      • Passwords are only accessible to the administrator of the database
    • There are restrictions on user passwords to prevent users from using weak passwords
      • Passwords must be at least 10 characters
      • Passwords must be less than 72 characters
      • Passwords must contain at least an uppercase letter, a lowercase letter, and a number
      • There are no restrictions on special characters in the password
  • Password are salted using a pseduorandom salt (works, but has yet to be integrated)
    • All user-inputted information is verified on the client and server
      • On the client I am using a large amount of JS I wrote
      • On the server I am using HTML Purifier
  • Brute forcing is impossible due to automatic account lockouts after 3 tries
  • All account login times and locations are stored
  • Only secure secret questions are used
